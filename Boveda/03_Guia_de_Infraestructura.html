4. Como se Actualiza el Sistema

Actualizaciones del Frontend

Cualquier cambio visual en la VCard, Check-In o Admin se actualiza mediante GitHub + Vercel:

  1. Se modifica el codigo fuente en el repositorio
  2. Se ejecuta git push al repositorio en GitHub
  3. Vercel detecta el cambio automaticamente y despliega en segundos
La gerente del spa no necesita ejecutar estos comandos. Los contenidos editables (estado, personal, comisiones) se manejan 100% desde el Command Center sin necesidad de despliegues.

Actualizaciones del Backend

Si se necesita modificar la logica del backend (por ejemplo, agregar un nuevo modulo):

  1. Abrir el editor de Google Apps Script vinculado a la hoja de calculo
  2. Modificar el codigo
  3. Guardar y crear un nuevo despliegue (Deploy > New deployment > Web app)
  4. Actualizar la URL del API en los archivos del frontend si cambio

5. Dominio Personalizado

Configuracion CNAME (Opcional)

Si el cliente desea usar un subdominio personalizado (por ejemplo: vcard.spaclarodeluna.mx):

  1. Acceder al panel DNS del dominio spaclarodeluna.mx
  2. Crear un registro CNAME apuntando vcard a cname.vercel-dns.com
  3. Agregar el dominio en el dashboard de Vercel (Settings > Domains)
  4. Vercel emitira el certificado SSL automaticamente
